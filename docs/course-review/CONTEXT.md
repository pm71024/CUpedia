# 课程测评（Course Review）

给中大学生的课程口碑工具：浏览真实课程目录（约 4800 门，与[课程技能树](../course-tree/CONTEXT.md)同一份 `courses` 数据），对课打分、匿名写评论、给评论点赞。**读匿名公开、写需 CUHK 登录**。它**不是**选课系统，也**不做**毕业审计——只沉淀"这门课怎么样"的众包口碑。

本文件只是术语表，不记录实现。

## Language

**综合推荐指数（Aggregate rating）**:
一门课所有评分的均值（0–10），课程列表与详情页展示的口碑分。**没有任何评分时为"暂无"（null），不等于 0**。
_Avoid_: 评分、打分（那是单个用户的动作，不是聚合值）。

**评分（Rating）**:
单个登录用户给一门课打的 0–10 分。可反复更新，但同一门课两次改分之间有**冷却**（防刷分）；聚合进「综合推荐指数」。
_Avoid_: 把评分等同于「综合推荐指数」；与「评论」混用——评分是数字、评论是文字，各自独立。

**评论（Review）**:
登录用户对一门课发表的一段文字，**匿名**呈现（不显示作者）。归属某个 User（用于权限判定），但对读者不暴露身份。
_Avoid_: 评价（易与「评分」混）；把评论当 wiki 页面修订。

**点赞（Like）**:
用户对某条「评论」的认可，**按用户去重**（同一人重复触发＝取消），展示总数。
_Avoid_: 收藏、关注。

**撤回（Withdraw）**:
移除一条「评论」——**作者本人或管理员**可为之，硬删除（连带其「点赞」）。
_Avoid_: 与 wiki 的软删除/恢复混为一谈——评论撤回不可恢复。

**学科（Subject）**:
课程的学科归类 = 真实 `subject` 列（CSCI / ACCT …，即课号前四位学科码）。本上下文的课程分类维度就用它。
_Avoid_: 院系 / Faculty / 学院——课程数据里没有院系字段，本上下文刻意不引入该维度；「专业大类」由[分院帽](../college-picker/CONTEXT.md)拥有，且那是通勤名次口径，与选课无关。

## Flagged ambiguities

- **课号（Course code）** 是与[课程技能树](../course-tree/CONTEXT.md)共享的锚点：技能树的「节点」与测评的一门课指向同一课号（如 CSCI3150），但两个上下文各存各的数据、MVP 不互链。
