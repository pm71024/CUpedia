# 分院帽（College Picker）

给 CUHK 新生用的书院志愿推荐工具：选专业大类 + 最看重的三个因素 + 想避开的因素，
按内置的名次与权重算出九所书院的志愿排序。面向用户叫「分院帽」，代码与 URL 用 `college-picker`。

本文件只是术语表，不记录实现。

## Language

**分院帽（College Picker）**:
本功能的规范名。面向用户的名字叫「分院帽」（呼应"分学院"的梗，新生向、游戏化调性）；
代码标识与 URL 段用 `college-picker`。
_Avoid_: 在中文语境里直接写 "College Picker"；"书院推荐器"（口语可，规范词用 分院帽）

**书院（College）**:
CUHK 九所住宿制书院的**类别词**（崇基、新亚、联合、逸夫、晨兴、善衡、敬文、伍宜孙、和声），
提供住宿与通识教育。分院帽推荐排序的对象。谈"类别/概念"时一律用「书院」。
_Avoid_: 用光秃秃的「学院」指代"书院"这一类别（见下方歧义）。

**专业大类（major_group）**:
工 / 理 / 商 / 社科 / 文 五个粗分桶，**仅用于查"上课通勤"名次**（不同专业主要上课地点不同，
通勤便利度因此不同）。
_Avoid_: 与课程技能树的「主修（Major）/主修树」混用——那边细到具体 programme，这里只有 5 个桶。

**志愿（Volunteer choice）**:
分院帽的输出 = 九所书院从**第 1 到第 9 的完整排序**（第一志愿、第二志愿……）。
不是"推荐一所"，而是给出整串报读优先次序。
_Avoid_: 把输出理解成"选一所书院"；与技能树的「构筑（Build）」混淆。

**看重因素 / 加分因素（Priority factor）**:
用户挑选的最看重的三个因素（可选：上课通勤、住宿环境、保宿机会、交换机会），
依次加权 **5 / 3 / 2** 计入基础分。
_Avoid_: 与"避雷因素"混为一谈——一个是想要的，一个是想避开的。

**避雷因素（Avoid）**:
用户想避开的属性（有书院 FYP、有宗教元素、有入学面试、有入学笔试）。命中的书院
**不会被剔除**，只是被尽量压到志愿末尾（加一个大额惩罚分）。
_Avoid_: 说成"硬性过滤/排除"——命中只是降权后移，书院仍在列表里。

**名次（rank）**:
某所书院在某个因素上的 **1–9 排名，1 为最优**。基础分由名次加权求和，**分越低越靠前**。
_Avoid_: 与最终的"志愿序号"混淆——名次是单因素的输入，志愿是综合后的输出。

## Flagged ambiguities

- **「学院」是重载词，须区分三义**：
  1. **书院个别成员的专有名后缀** —— "崇基学院"就叫这名字，**保留原样**，不得改写成"崇基书院"。
  2. **院系 / Faculty** —— "工程学院""理学院"，是学术单位，与住宿书院无关。
  3. （错误用法）用裸「学院」泛指"书院这一类"。

  规则：**作为类别词，住宿这一类一律说「书院（College）」，学术单位一律说「院系 / Faculty」，
  绝不用裸「学院」；但专有名保留原样。**

## 示例对话

> **甲**：分院帽是按专业推荐书院的吗？
> **乙**：不完全。它按你最看重的三个因素（通勤、住宿环境、保宿、交换）排书院志愿。专业大类只在算"上课通勤"名次时用到——工科和文科上课地点不同，通勤便利的书院就不同。
> **甲**：那这个"专业大类"是不是就是技能树里的主修？
> **乙**：不是。技能树的主修细到具体 programme；分院帽的专业大类只有工/理/商/社科/文五个桶，只为查通勤名次。两者别混。
