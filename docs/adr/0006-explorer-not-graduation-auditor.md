# 技能树是探索器，不是毕业审计器

本产品把选课游戏化，面向新生探索 +（Phase 2）高年级构筑分享，**不做毕业审计**。因此主修要求的**类目是展示分组**，带"软"的每类/总学分预算（只提示"还差 N 门 / 已满"），**不强校验**"任选 N 门""3000 级两门"这类规则。

唯一的**硬约束**集中在**严格模式**（从大一逐学期点亮）：课只能在与其开课季节匹配的学期点亮（"上"= 秋/Term 1，"下"= 春/Term 2，忽略学年与暑期）、每学期学分上限（默认 ~18）、先修顺序（不得点在其先修之前）；外加**等价组互斥**（多选一，选中后兄弟置灰）。这些反映的是"不可能/非法的选择"，而非毕业规则。**自由模式**一概不强制（先修边只连不拦，只受主修总学分软上限约束）。

先修硬拦基于把 `requirements` 解析出的**课号布尔逻辑**（OR 任一满足即解锁）；含 "or equivalent / or by permission" 等旁路的，降级为黄色警告不硬拦。

## Consequences

- "点满"的构筑**不等于**满足毕业要求——这是有意为之。
- 若日后要做真正的学位审计，需要新增基于 handbook 全量规则（GE / University Core / basket 细则）的校验层，是另一项决策。
