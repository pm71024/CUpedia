# 课程技能树（Course Skill Tree）

把 CUHK 选课重新包装成"技能树加点"的游戏化探索工具。主要受众是**新生**——用逛技能树的方式看"有什么课能选、课与课之间怎么连"；叠加社区层（Phase 2），让**高年级学生**把自己真实选过的课作为"构筑（Build）"分享。它**不是**毕业学分审计器；数据来自课程数据集 + handbook（来源见 ADR 0005 与 #157），MVP **不与 wiki 互链**。

本文件只是术语表，不记录实现。

## Language

**节点（Node）**:
技能树的原子单位 = 一门真实的具体课程（如 CSCI1130）。新生看到的就是真课；构筑必须能落到具体课程上。
_Avoid_: 把节点当成抽象的"技能/主题"——本项目刻意选了"节点 = 单门课"。

**自由模式（Free mode）**:
一种加点模式：只受"学分上限"（= 该主修总学分，作软进度线）约束，**忽略**开课学期与先修顺序（先修边只连不拦）。用于纯探索"我想学这些"。
_Avoid_: 把它当成无约束——它仍有学分上限。

**严格模式（Strict mode）**:
一种加点模式：从大一**逐学期推进**点亮课程（每个学期一组、一种颜色），强制 (a) 课只能在与其开课季节匹配的学期被点亮——"上"学期 = 秋 / Term 1，"下"学期 = 春 / Term 2，两季都开的课不限（**忽略具体学年与暑期**，由 `terms` 字段判定季节）；(b) 每学期学分上限（默认 ~18，可调）；(c) 先修顺序——不得把课点在其先修之前。学制按主修正常年限（如 4 年 8 学期）；全部点完且满足软要求时整棵树视为"点亮"。
_Avoid_: 与"自由模式"混用同一套约束；把"上/下"学期当成锁定学年（同一门 Term 1 课在大一上与大三上都可点）。

**主修树（Major tree）**:
技能树的范围单位 = 一个主修（Major/Programme）。入口是"选一个主修"，进入后看到该主修的课程树。辅修（Minor）作为独立 tab（待定）。
_Avoid_: 以 subject/系（如"CSCI 系全部课"）当作树的范围——本项目刻意选了"按主修"。

**类目（Requirement category）**:
主修要求里的一个分组，带"应修学分数"。每个类目呈三种形态之一：(a) 写死的必修课；(b) "X 或 Y"二选一；(c) "从一篮子任选 N 门"。技能树把类目当**展示分组 + 学分预算**，只提示"还差 N 门/已满"，**不强校验**"任选 N 门""3000 级两门"等规则（本产品是探索器，非毕业审计器）。
_Avoid_: 与课程节点混为一谈——类目是节点的分组容器。

**构筑（Build）**:
一次"加点"的产物：一个**已点亮课程的集合**，可选附带"每门课的目标学期"（严格模式或主动排期时填充）。归属于某个 User；高年级分享的就是它。
_Avoid_: 把 Build 等同于"逐学期计划"——学期信息是可选的，不是必填。

**层级（Layer）**:
节点的纵向分层依据：优先按先修连边；无明显先修则按课号首位（1/2/3/4 ≈ 年级）分层，可整棵默认用课号层级。
_Avoid_: 把层级当成"修读顺序强制"——它主要是布局/阅读次序。

**先修边（Prerequisite edge）**:
节点间的有向连边。源数据是课程 `requirements` 字段，但该字段**混装**先修（`Pre-requisite: …`）、排斥（`Not for students who have taken …`）与备注（豁免/院系限制），须先**分句归类**，再把先修句解析成**课号布尔逻辑**（AND/OR，简写如"or 1130"要补全前缀）。先修可**跨科目、指向主修树外**的课。
_Avoid_: 把整个 `requirements` 当先修；把含 "or equivalent/或获许可" 的旁路当精确硬先修（这些降级为警告，见严格模式）。

**等价组（Equivalence group）**:
一组**互斥、同角色**的课程（由排斥数据推出），如 9 门"编程入门"。树里渲染为同层并列的"多选一"：选中一门即连边、其余**置灰不可选**；下游先修指向组内任一即满足。
_Avoid_: 把等价组当成"必须全修"的类目——它是"选其一"。
